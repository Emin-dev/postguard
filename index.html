Honest scope — read this first

This category is real, and it's crowded with good free tools

Metadata-scrubbing tools are not a novel idea. Free, well-reviewed competitors already do this well — Scrambled Exif (54,000+ downloads, 4.04/5 on F-Droid) and Chrome's own built-in Metadata Remover extension (4.6/5) both strip EXIF data for free, and have for years. If you just want EXIF gone, either of those is a perfectly good choice and you don't need this page.

The one honest reason Postguard exists: none of the popular free scrubbers also let you manually redact parts of the image (a face, a license plate, a street sign) in the same flow. Postguard bundles metadata-scrub and redaction together so you can do both in one place before sharing a photo. That's a real, if modest, convenience — not a breakthrough. Build this expecting portfolio and utility value, not meaningful revenue; the market evidence for a paid version of this exact idea is weak.

100% local — verify it yourself

Every operation below — reading your file, parsing its metadata, stripping it, drawing redaction boxes, rendering the cleaned output — happens with the browser's local File, ArrayBuffer, and <canvas> APIs only. This app makes zero network requests. There is no fetch, no XMLHttpRequest, no analytics beacon, and no server component at all — index.html is served as a static file and every script in js/ is plain client-side JavaScript. Open your browser's Network tab while using this page and you'll see nothing leave your machine. View source, read js/exif.js, js/pdfmeta.js, and js/redact.js directly if you want to confirm it yourself — that's the point of shipping this with no build step and no bundler.

What's real in this repo

PartCoverage
JPEG/EXIF parser (js/exif.js) REAL — real JPEG marker-segment walking, real TIFF/IFD binary parsing. Covers: GPSLatitude/GPSLongitude (+ N/S/E/W refs), Make, Model, Software, DateTime, DateTimeOriginal, Artist, Copyright. Other EXIF tags aren't displayed, but stripping removes the whole APP1 segment regardless.
PDF metadata parser (js/pdfmeta.js) REAL — real trailer//Info-dictionary text parsing plus embedded XMP dc:creator/dc:title extraction. Covers: Author, Title, Subject, Keywords, Creator, Producer. Does not support encrypted PDFs or PDF 1.5+ compressed cross-reference streams.
Redaction engine (js/redact.js) REAL — destructive pixel overwrite (solid fill or box blur) on real ImageData, Node-tested against synthetic pixel buffers.
